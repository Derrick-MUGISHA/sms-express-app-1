# Student Management System API (sms-express)

A robust backend API for a Student Management System built using **Node.js, Express.js, Prisma ORM**, and **MongoDB**. The API provides fully-featured functionality for user authentication, course administration, student enrollments, and attendance tracking.

## Functionality
- **Authentication & Authorization**: 
  - User Registration with OTP email verification (Nodemailer).
  - JWT-based login (Access and Refresh tokens).
  - Role-based access control (`STUDENT`, `SUPERVISOR`).
- **User Management**:
  - Secure password hashing using bcrypt.
  - Automatic seeding of supervisor accounts.
- **Course Administration**:
  - Creation, retrieval, updating, and deletion of courses.
- **Enrollment System**:
  - Securely enroll students into specific courses.
- **Attendance Tracking**:
  - Track student attendance per course under the supervision of authorized instructors/supervisors.
- **API Documentation**:
  - Comprehensive automatic API documentation provided by Swagger UI.

## Technology Stack
- **Backend Framework**: Node.js & Express.js
- **Database Architecture**: MongoDB
- **ORM**: Prisma Client
- **Authentication**: JSON Web Tokens (JWT) & bcrypt
- **Mailing service**: Nodemailer (SMTP)
- **Containerization**: Docker & Docker Compose
- **API Documentation**: Swagger (swagger-jsdoc, swagger-ui-express)

## ⚙️ How to Set It Up

### 1. Prerequisites
Ensure you have the following installed to run this project natively or via Docker:
- [Node.js](https://nodejs.org/en/) (v16+ recommended working locally)
- [Docker](https://www.docker.com/) & Docker Compose
- A [MongoDB](https://www.mongodb.com/) cluster or local DB instance

### 2. Environment Variables
Create a `.env` file in the root directory and provide the necessary variables representing your environment setup:
```env
# Database Configuration
DATABASE_URL="mongodb+srv://<username>:<password>@cluster0.db.net/SMS?appName=Cluster"

# JWT Token Secrets
JWT_ACCESS_SECRET="your_access_secret_string"
JWT_REFRESH_SECRET="your_refresh_secret_string"

# Allowed CORS Origins
ALLOWED_ORIGINS="http://localhost:3000,http://yourdomain.com"

# Application Port
PORT=3000

# SMTP / Email Configuration for OTP Verification
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your_email@gmail.com
SMTP_PASS=your_email_app_password

# Default Supervisor Seeding details
SUPERVISOR_EMAIL=admin@example.com
SUPERVISOR_PASSWORD=adminpassword123
```

### 3. Local Setup (Without Docker)

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Generate the Prisma Client:**
   ```bash
   npx prisma generate
   ```

3. **Push the Schema to MongoDB:**
   ```bash
   npx prisma db push
   ```

4. **Seed the database (Creates the default super admin / supervisor):**
   ```bash
   npm run prisma:seed
   ```
   *(Note: Or run manually via `node prisma/seed.js`)*

5. **Start the Development Server:**
   ```bash
   npm run dev
   ```
   *(The server will start at `http://localhost:3000`)*

### 4. Docker Setup (Recommended)

To run the application entirely isolated via Docker, ensuring all environment keys are correctly set in the `.env` file:

1. **Build and spin up the containers:**
   ```bash
   docker-compose up -d --build
   ```

2. **Run database scripts within the container:**
   ```bash
   docker-compose exec app npx prisma generate
   docker-compose exec app npx prisma db push
   docker-compose exec app node prisma/seed.js
   ```
